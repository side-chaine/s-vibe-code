# ⚡ QUICK-START: Подключи S-VC к своему проекту за 5 минут

## 1. Склонируй S-VC Core

```bash
git clone https://github.com/side-chaine/s-vibe-code.git
```

## 2. Скопируй AGENTS.md в корень проекта

```bash
cp s-vibe-code/AGENTS.md ./AGENTS.md
```

Это единственный обязательный файл. OpenCode/ClaudeCode читают `AGENTS.md` в корне проекта и настраивают агента по нему.

## 3. Скопируй чартеры

```bash
mkdir -p agents/charters
cp s-vibe-code/charter-007.md agents/charters/
cp s-vibe-code/charter-009.md agents/charters/
cp s-vibe-code/charter-operator.md agents/charters/
```

Чартеры — детальные инструкции для каждого агента. Они не обязательны для запуска, но без них агенты не знают протокола.

## 4. Настрой модели агентов (ВАЖНО!)

S-VC использует **разные модели для разных ролей**:

| Агент | Модель | Зачем |
|-------|--------|-------|
| **001** (Architect) | 🏋️ **Тяжёлая** (Sonnet/GPT-4o) | Принимает архитектурные решения |
| **007** (Scout) | 🪶 **Лёгкая** (default) | Собирает контекст, упаковывает |
| **009** (Verifier) | 🪶 **Лёгкая** (default) | Проверяет код, читает файлы |
| **Operator** | 🪶 **Лёгкая** (default) | Слепо исполняет TC |

Скопируй agent-файлы:

```bash
mkdir -p .opencode/agent
cp -r s-vibe-code/.opencode/agent/* .opencode/agent/
cp s-vibe-code/opencode.json .
```

**Что происходит под капотом:**
- 007 работает на твоей default-модели (лёгкая → дёшево)
- Когда 007 вызывает `task(subagent_type="001")` — OpenCode автоматически спавнит 001 на **Sonnet** (тяжёлая)
- 007, 009, Operator используют default-модель

**Важно:** Убедись что у тебя есть доступ к `claude-sonnet-4-20250514` (или замени в `.opencode/agent/001.md` на свою сильную модель).

## 5. Настрой AGENTS.md под свой проект

Открой `AGENTS.md` и замени `[PROJECT]` на название своего проекта:
- В стартовом контексте
- В путях к справочникам
- В описании проекта

Если у тебя есть архитектурная документация — добавь ссылки в раздел "Справочники".

## 6. Напиши первую задачу

```markdown
# Задача: [что сделать]
# Домен: [какая часть проекта]
```

Когда OpenCode/ClaudeCode запустится — 007 (Scout) распознает задачу и начнёт цикл.

## Факультативно: создай файлы контекста

S-VC использует пакеты контекста — файлы, которые 007 читает при старте.

Рекомендуемая структура:

```
your-project/
├── AGENTS.md              ← скопирован из S-VC
├── agents/
│   ├── charters/
│   │   ├── charter-007.md ← скопирован
│   │   ├── charter-009.md ← скопирован
│   │   └── charter-operator.md ← скопирован
│   └── packs/
│       ├── 000-bootstrap.md    ← создай сам: краткий стартовый пакет
│       └── 000-ref-arch.md     ← создай сам: архитектурная карта
└── context/
    └── tasks/             ← сюда 007 будет класть пакеты задач
```

### 000-bootstrap.md (пример)

```markdown
# 🔺 BOOTSTRAP [PROJECT]
## Краткий стартовый пакет

## 1. ПРОЕКТ
[PROJECT] — [описание проекта, стек, ключевые решения]

## 2. АРХИТЕКТУРА
[краткая архитектурная модель]

## 3. FROZEN ZONES ([проект])
[список того, что нельзя менять без Architect]

## 4. OWNERSHIP
[кто чем владеет]
```

## Готово!

S-VC работает. Теперь:
- Задачи → проверка → код → верификация — всё по протоколу
- Никакой бюрократии — просто `AGENTS.md` в корне
- Если что-то пошло не так — `Error = Stop`, агент доложит
