# 🎯 S-VC Core — Side_chaine_Vibe_Code

**S-VC (Side_chaine_Vibe_Code)** — открытая система агентского governance для проектов, которые пишутся с AI-агентами (vibe coding, chat-oriented programming, AI-assisted development).

> **Один AGENTS.md — и твой проект говорит с AI-агентами на одном языке.**

---

## Зачем это нужно

Когда проект пишут AI-агенты, возникает хаос:
- 🤖 Разные агенты не знают контекста друг друга
- 📝 Кто-то пишет код, кто-то его ломает
- 🧠 Решения принимаются без архитектурной карты
- 🔄 Один агент не проверяет работу другого

S-VC решает это **протоколом, а не тулом**. Ты не устанавливаешь сервер, не настраиваешь БД. Ты просто кладёшь `AGENTS.md` в корень проекта — и система работает.

---

## Как это работает

```
S-VC — 3 фазы:

╔════════════════════════════════════════════════════════╗
║  PHASE 1 — RECON                                      ║
║  Задача → Scout собирает контекст → Architect решает  ║
╚════════════════════════════════════════════════════════╝
                      ↓
╔════════════════════════════════════════════════════════╗
║  PHASE 2 — EXECUTION                                  ║
║  Scout собирает инструкцию → Operator применяет код   ║
╚════════════════════════════════════════════════════════╝
                      ↓
╔════════════════════════════════════════════════════════╗
║  PHASE 3 — VERIFICATION                               ║
║  Scout проверяет → Verifier верифицирует → COMPLETE   ║
╚════════════════════════════════════════════════════════╝
```

### 6 ролей

| Роль | Кодовое имя | Делает |
|------|-------------|--------|
| **Ты** | Product Manager | Ставишь задачи |
| **Scout** | **007** | Разведка, упаковка контекста, DOC-CHECK |
| **Operator** | — | Слепое исполнение технических задач |
| **Verifier** | **009** | Независимая верификация результатов |
| **Architect** | **001** | Архитектурные решения, governance |
| **Stress-Tester** | **002** | Контрразведка. Атакует предположения |

## Принципы

- ❄️ **Изоляция** — Scout и Operator работают в разных инстансах
- 🎯 **1 TC = 1 правка** — атомарные изменения
- 📝 **Doc-Aware** — каждый фикс сопровождается проверкой документации
- 🔍 **009** — независимая верификация, защита от confirmation bias
- 🚫 **Frozen zones** — без Architect'а не трогать
