## 3. ПРАВИЛА

### RECON FIRST
Перед любым edit — grep. Доказать что строка существует:
```
grep confirmed at line X: [строка]
```
Если grep не находит anchor → **STOP**. Доложить 007.

### MAX 2 FILES
За одну задачу — максимум 2 файла. Если в MICRO-PACK больше — **STOP**, спросить 007.

### NO CLEANUP
Менять ТОЛЬКО то что в TC. Ничего рядом не трогать.

### PROVE-IT
Не "it exists", а "grep confirmed at line X".

### ANCHORS
Найти anchor-строки ДО и ПОСЛЕ изменений. Подтвердить что контекст совпадает.

### PRESERVE STYLE
Сохранять табы/пробелы, комментарии, форматирование окружающего кода.

### SELF-CHECK
Перед применением:
- Только target lines изменены?
- Скобки сбалансированы?
- Отступы не сломаны?

### VERIFY TYPES
```
tsc --noEmit
```
Если ошибки → **STOP**. Доложить 007 с полным выводом ошибок.

### VERIFY TESTS
```
test --related
```
Если падают → **STOP**. Доложить 007 с логом падений.

### ERROR = STOP
Любая ошибка → STOP. Не чинить самому. Доложить 007:
```
❌ TC-XXX: ОШИБКА
Причина: [что пошло не так]
Детали: [логи, grep, tsc output]
```
